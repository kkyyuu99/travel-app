# Claude 세션 지침 — Travel PWA

> 이 파일은 Claude Code가 이 디렉토리에서 작업할 때 자동으로 로드된다.
> 여기 적힌 규칙은 **사용자가 별도로 지시하지 않는 한 모든 세션에 적용**된다.

---

## 0. 가장 중요한 단 하나의 규칙

**작업 시작 전 [WORKTHROUGH.md](WORKTHROUGH.md)를 먼저 읽어라.**
작업 종료 전 그 파일의 §9 (세션 로그) 맨 아래에 새 항목을 한 개 append 해라.

이것이 세션 간 컨텍스트 인계의 유일한 메커니즘이다. 빠뜨리면 다음 세션이 같은 질문을 다시 하게 된다.

---

## 1. 세션 시작 시 (Pre-flight)

1. **`WORKTHROUGH.md` §1~§3을 읽는다** — 현재 버전, 파일 구조, 데이터 모델 파악.
2. **`WORKTHROUGH.md` §9 마지막 3~5개 항목을 읽는다** — 직전 세션이 무엇을 했고 무엇을 남겼는지.
3. **`WORKTHROUGH.md` §7 (TODO)을 본다** — 미해결 이슈가 사용자 요청과 관련 있으면 먼저 언급.
4. 그 다음 사용자 요청에 답한다.

읽었음을 사용자에게 보고할 필요 없다. 조용히 읽고 작업에 반영.

---

## 2. 세션 로그 작성 (Post-work — 의무)

작업이 다음 중 하나라도 해당하면 `WORKTHROUGH.md` §9에 항목을 추가한다:

- 코드 변경 (1줄이라도)
- 새 파일 / 디렉토리 추가
- 외부 의존성 / API 키 / 환경 변경
- 새 의사결정 (왜 A 대신 B를 택했는지)
- 디버깅 끝에 알게 된 비자명한 함정
- 사용자의 작업 의도/스코프가 바뀜

**해당 안 됨**: 단순 질문 답변, 파일 읽기만, 사용자가 "안 할래"로 끝낸 탐색.

### 포맷

```markdown
### YYYY-MM-DD — 한 줄 요약 (40자 이내)
- 무엇을 했나: 커밋 해시 / 파일:라인 / 함수명 인용
- 왜 했나: 사용자가 명시한 이유 또는 발견한 문제
- 다음 세션이 알아야 할 것: 잔여 작업, 함정, 임시 우회로
```

**날짜는 절대값으로** ("어제" 금지, `2026-05-22`).
**앞 항목 절대 삭제·수정 금지** — append only.

§7 (TODO)에 추가한 항목이 있으면 세션 로그에서 그쪽을 명시적으로 가리킨다.

---

## 3. 의사결정이 생기면 §8에 박아라

"왜 X 대신 Y를 했나"가 비자명하면 `WORKTHROUGH.md` §8 (의사결정 로그)에 한 줄 추가.
이건 코드 주석이나 커밋 메시지로는 안 남는다 — 다음 세션이 같은 갈림길에서 헤매지 않게 하는 게 목적.

예시:
> - **vNN에서 React 도입 안 함** — 빌드 도구 없는 원칙 유지, 5000줄 단일 HTML 감수.

---

## 4. 이 프로젝트의 도메인 규칙

### 빌드 도구 금지
- npm/webpack/vite 등 도입 금지. 순수 HTML/CSS/ES2020.
- 의존성 추가 필요하면 CDN 링크 + SW 캐시. 사용자에게 먼저 확인.

### 버전 / 캐시 관리
- 코드 변경하면 `sw.js`의 `CACHE_VERSION` 을 올린다 (`v21` → `v22`).
- 동시에 `PRECACHE_URLS`의 `?v=NN` 도 같은 번호로 맞춘다.
- `index.html` 안의 `<script src="js/xxx.js?v=NN">` 도 같이 맞춘다.
- 이 3곳이 어긋나면 캐시 갱신이 안 됨 — 핵심 함정.

### 로컬 우선, 클라우드는 미러
- IndexedDB가 source of truth.
- 클라우드(Supabase) 연동을 추가할 때 로컬-only로도 동작해야 함 (로그인 안 한 사용자).
- 비행기 모드 동작이 1순위 가치.

### Supabase 키
- `supabase-config.js` 의 `publishableKey` 는 공개 OK (RLS 보호).
- `service_role` 키는 **절대 커밋·노출 금지**. 발견 시 즉시 사용자에게 경고.

### `_archive/` 건드리지 마라
- v1 시절 백업. 회귀 비교용. 삭제·수정 금지.

### 커밋 메시지 컨벤션
- `vNN: 한 줄 요약` 형태 (현재까지 v1~v21이 이 패턴).
- 새 마이너 버전이면 번호를 올린다.

---

## 5. 사용자 응답 스타일

- 한국어로 답한다 (사용자가 한국어로 요청).
- 단답형 질문엔 단답. 헤더/표 남발 금지.
- 변경한 파일은 마크다운 링크로 인용: `[index.html:120](index.html#L120)` 같은 식.
- 코드 변경 후 "변경사항 요약" 섹션을 자동으로 길게 늘어놓지 말 것 — diff가 이미 있음.

---

## 6. 절대 하지 말 것

- `WORKTHROUGH.md`의 과거 세션 로그 항목 수정·삭제
- `_archive/` 폴더 수정
- `service_role` 키나 비밀 토큰을 어떤 파일에든 쓰기
- 빌드 도구 / 프레임워크 도입 (먼저 사용자에게 명시적 허락 받기)
- 사용자가 요청 안 한 리팩토링 (5000줄 `index.html`이 거슬려도 분할하지 마라)
- `git push --force`, `git reset --hard`, 미커밋 파일 위에 `git checkout .` — 명시 허락 없이 금지
